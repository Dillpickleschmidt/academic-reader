# Explore authenticated SSR for the home route only

## What to explore

Evaluate whether Academic Reader can server-render only the home route with authenticated content while keeping the rest of the app as the current Vite/Solid SPA.

The motivation is to avoid the home page rendering a neutral/session-pending shell and then switching after client hydration when the Reader is already signed in.

## Candidate approach

Use the existing Hono API server as a narrow SSR entrypoint for `/`:

- Read Better Auth cookies from the request.
- Resolve the current Better Auth session/token through Convex Better Auth HTTP actions or server utilities if available.
- Query Convex for the current Reader and Document library.
- Render the home route HTML with Solid server rendering.
- Serialize minimal initial auth/library state for hydration.
- Hydrate the existing Solid client without mismatch.
- Send `Cache-Control: no-store` for authenticated SSR responses.

## Constraints

- Keep SSR scoped to the home route unless a stronger reason appears.
- Do not migrate to TanStack Start as part of this issue.
- Do not add TanStack Query just for this spike.
- Do not make the app-wide routing model more complex unless the home-route-only path proves too awkward.
- Preserve unauthenticated upload/configuration entry behavior.

## Questions to answer

- Can Better Auth + Convex expose a clean server-side session/token path from Hono without custom brittle cookie parsing?
- Can Solid SSR for only `/` coexist cleanly with the generated TanStack Solid Router SPA bundle?
- How much hydration state is required to avoid flicker and mismatches?
- Does this reduce meaningful UX pain compared with a neutral pending shell?
