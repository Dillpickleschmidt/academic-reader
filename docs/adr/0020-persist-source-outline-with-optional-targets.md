# Persist source outline entries with optional targets

Academic Reader persists the full PDF-provided outline as Table of Contents Entries, even when an entry has no resolved in-app target or points outside the processed Page range. The API app, not a Conversion Model worker, extracts this Source Document metadata so Page Labels and outline entries are independent of Marker/LightOnOCR/Chandra behavior. This preserves source-provided navigation structure without inventing links: entries with resolved targets can navigate in the Reader View, while unresolved or unavailable entries remain visible but disabled. Coordinate destinations keep their resolved Source Point for Debug Overlay evidence, and Block targets are derived by choosing a nearby, unambiguous Block top edge on the same Physical Page rather than by text or heading inference. TOC navigation moves the Reader View only; Source View remains stable unless the reader uses Source/Reader cross-view controls.
