# Stage Narration before audio

Academic Reader stages Narration after Reader View readiness instead of generating audio directly from Blocks. When Narration is enabled in Processing Configuration, the app marks Inline Citations in Block HTML, derives Narration Candidates, persists each Block's Narration Eligibility and Narration Preparation, generates a compact unstructured Narration Guide from eligible Blocks only, creates Narration Text, and then generates Narration audio as each Block's Narration Text becomes available.

Reader View readiness remains independent of Narration. Once Pages and Blocks are ready, the Document may become `ready` or `readyWithWarnings`; later Narration failures do not mutate that earlier status. Narration work still belongs to the same Processing Run and remains visible in the Document's Processing Event sidebar.

Narration Eligibility and Narration Preparation are persisted together on each Block. Eligible Blocks store strict Narration Preparation tags such as `plain`, `inline-citation-cleanup`, `inline-math`, and `equation-explanation`; ineligible Blocks store a strict reason. Narration Text is stored only on eligible Blocks once prepared. The Document stores an unstructured `narrationGuide` string used as shared context for per-Block or small-batch rewrites.

Narration AI work uses Groq first, with explicit task model settings for Eligibility Review, Narration Guide generation, and Narration Text rewrite. The app does not fail startup when Groq configuration is missing; instead, the affected Narration phase emits a failed Processing Event and does not patch all Blocks with synthetic per-Block failures. Non-Groq providers can be configured later, but v1 fails the Narration task that needs AI rather than carrying unused provider abstraction.

Narration uses phase-specific Processing Event types instead of generic narration lifecycle events because the phases are fixed and reader-visible: candidate extraction, eligibility review, guide generation, rewrite, and audio generation. The generic `narration.started`, `narration.progress`, `narration.completed`, `narration.warning`, and `narration.failed` event types are replaced by strict phase-specific families: `narration.candidates.*`, `narration.eligibility.*`, `narration.guide.*`, `narration.rewrite.*`, and `narration.audio.*`. Rewrite calls process small batches by default, with per-Block retry fallback when a batch output is incomplete or invalid.
