# Conversion adapters preserve block evidence

Academic Reader uses an adapter for each conversion model and produces one unified output schema. Schema fields are required by default; a field becomes optional only when at least one supported model cannot provide it or local processing cannot derive it reliably. Block content is HTML-first: adapters must produce `contentHtml`, deriving it from Markdown when needed, while `contentMarkdown` remains optional. Source Geometry is optional for Blocks because LightOnOCR can produce readable Markdown but does not provide text-block geometry; it may still provide geometry for image/figure regions. The LightOnOCR adapter still parses Markdown into semantic Blocks for the Readable View, attaching Source Geometry only where LightOnOCR actually returned it. The Debug Overlay shows available Block Evidence without forcing unsupported fields or inventing fake values.
