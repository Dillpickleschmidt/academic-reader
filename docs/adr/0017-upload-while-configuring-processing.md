# Upload while configuring processing

Academic Reader starts uploading a selected source document immediately to object storage under a random temporary upload ID while showing Processing Configuration. Temporary uploads live only in object storage until claimed; Convex records start when an authenticated reader confirms Processing Configuration and creates a Document. The configuration screen displays upload progress, and the action that starts the Processing Run remains disabled until the upload completes; this overlaps slow uploads with reader configuration without starting immutable processing work before the reader confirms choices. Unauthenticated visitors can upload/configure first and sign in only when they attempt to start processing.
